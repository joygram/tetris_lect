# 개념 실습 테스트 환경 가이드

## 📖 이 가이드가 필요한 이유

**기초 개념 학습**(변수, 배열, 메소드 등)을 할 때는 **테트리스 게임 씬과 분리된 전용 씬**에서 실습하는 것이 좋습니다.

- **프로젝트(테트리스)와 상관없이** 개념만 단계적으로 연습할 수 있고  
- **씬이 단순해서** 실습 결과를 Console·화면으로 바로 확인할 수 있으며  
- **게임 씬을 건드리지 않으므로** 단계별 학습(1~8단계) 진행에 영향을 주지 않습니다.

**중요**: **왔다 갔다 하지 않습니다.**  
각 **개념 실습마다 전용 씬**을 만들고, **그 씬에서만** 해당 실습을 진행합니다.  
각 개념 문서의 실습 맨 앞에 **"이 실습 전용 씬 만들기 (환경 설정)"**이 있으므로, **그 문서만** 따라 하면 됩니다.

---

## 🎯 목표

- **개념별로 전용 씬** 하나씩 만들기 (예: ConceptTest_Variable, ConceptTest_Array, …) — **씬 이름은 Unity에서 한글이 지원되지 않으므로 영문으로 저장**하세요.  
- **한 문서에서** 환경 설정 → 실습까지 **한 흐름**으로 진행 (다른 문서로 이동하지 않음)  
- **씬 전환**으로 "개념 실습" ↔ "테트리스 게임" 구분하기  

---

## 1단계: 씬 저장 폴더 준비 (한 번만)

1. **Project** 창에서 **Assets** 안을 확인합니다.
2. **Scenes** 폴더가 있으면 그 안에, 없으면 **Assets** 아래에 **Scenes** 폴더를 만듭니다.  
   - 우클릭 → **Create** → **Folder** → 이름: `Scenes`

→ 이후 각 개념 문서의 **"이 실습 전용 씬 만들기"**에서 **File → Save As**로 이 폴더에 씬을 저장합니다.  
**참고**: Unity는 **씬 파일 이름에 한글을 지원하지 않습니다.** 파일 이름은 **영문**으로 저장하세요 (예: ConceptTest_Variable).

---

## 씬 만드는 방법 (한 번만 따라 하기)

**각 실습**을 시작할 때, 해당 개념 문서의 **"이 실습 전용 씬 만들기"**에서 **반드시 먼저 씬을 만든 뒤** 수행 과정을 진행합니다. 아래 순서대로 하면 됩니다.

### 1. 새 씬 만들기

1. 메뉴에서 **File**(파일) 클릭
2. **New Scene**(뉴 씬) 클릭
3. **Basic (Built-in)** 또는 **Empty**(엠프티) 중 하나 선택
4. **Create**(크리에이트) 버튼 클릭  
   → Hierarchy에 빈 씬이 열립니다.

### 2. 씬 저장하기

1. 메뉴에서 **File** → **Save As**(세이브 애즈) 클릭
2. 저장 위치에서 **Assets** 폴더 열기 → **Scenes** 폴더 선택  
   (Scenes 폴더가 없으면 위 **1단계: 씬 저장 폴더 준비**대로 먼저 만드세요.)
3. **파일 이름** 칸에 해당 문서에 적힌 **영문** 이름 입력 (예: `ConceptTest_Variable`, `ConceptTest_Array`)
4. **Save**(세이브) 버튼 클릭  
   → 상단 탭에 입력한 씬 이름이 보이면 저장된 것입니다.

### 3. 이 씬에서 실습 진행

이제 **같은 개념 문서**의 **"수행 과정"**을 따라, **이 씬에서만** 스크립트 만들기·오브젝트 추가·Play 확인을 하면 됩니다.

**이미 씬을 만들어 둔 경우**: **File** → **Open Scene**(오픈 씬) → **Assets/Scenes** 안에서 해당 씬 선택 → 열고 나서 수행 과정을 이어가면 됩니다.

---

## 2단계: 개념별 전용 씬 (각 실습 문서에서 진행)

**각 기초 개념 문서**(변수, 배열, 멤버와메소드, 게임루프, MonoBehaviour, 멤버변수와로컬변수 등)의 **🛠️ 실습**에는 맨 앞에 **"이 실습 전용 씬 만들기 (환경 설정)"**이 있습니다.

**진행 방법**:
1. 해당 개념 문서를 연다.
2. **"이 실습 전용 씬 만들기"** 절을 따라 **먼저 씬을 만든다.** (위 **씬 만드는 방법** 참고. 이미 만들었으면 **File → Open Scene**으로 그 씬을 연다.)
3. **같은 문서**에서 **수행 과정**을 따라 스크립트 작성 → 빈 오브젝트에 추가 → Play → Console 확인까지 **이 씬에서만** 진행한다.

**다른 문서로 가지 않습니다.** 환경 설정과 실습이 **같은 문서 한 흐름**에 포함되어 있습니다.

### 씬 이름 예시 (영문만 사용)

Unity는 **씬 파일 이름에 한글을 사용할 수 없습니다.** 아래처럼 **영문**으로 저장하세요.

| 개념 문서             | 전용 씬 이름 (Save As 시, 영문)     |
| --------------------- | ----------------------------------- |
| 변수                  | ConceptTest_Variable                |
| 배열                  | ConceptTest_Array                   |
| 멤버와 메소드         | ConceptTest_MemberMethod            |
| 멤버 변수와 로컬 변수 | ConceptTest_MemberLocal             |
| 게임 루프             | ConceptTest_GameLoop                |
| MonoBehaviour         | ConceptTest_MonoBehaviour           |
| 함수                  | ConceptTest_Function                |
| 클래스                | ConceptTest_Class                   |
| (기타 개념 문서)      | 해당 문서에 안내된 **영문** 씬 이름 |

---

## 3단계: 테스트용 스크립트 폴더 (선택)

개념 문서마다 실습 스크립트(VariableTest, ArrayTest 등)를 만들 때, **게임 스크립트와 구분**하려면:

1. **Project** → **Assets/Scripts** 선택  
2. 우클릭 → **Create** → **Folder**  
3. 이름: `ConceptTests` (또는 `테스트`)  
4. 개념 실습 시 만드는 스크립트는 이 폴더에 생성  

→ 나중에 "게임용 스크립트"와 "개념 실습용 스크립트"를 구분하기 쉬워집니다.

---

## 4단계: 씬 전환 요약

| 하려는 일               | 할 일                                                                                                                                                              |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **특정 개념 실습** 하기 | 해당 **개념 문서**를 열고 → "이 실습 전용 씬 만들기" 따라 씬 생성/열기 → 같은 문서에서 수행 과정 진행                                                              |
| **다른 개념 실습** 하기 | **해당 개념 문서**를 열고 → 그 문서에 안내된 **전용 씬**을 열고 → 그 문서의 수행 과정 진행                                                                         |
| **테트리스 게임** 하기  | **File → Open Scene** → **Tetris** (게임용 씬) 선택. **작업 씬 제차 확인** 필수 → [1단계 씬 이름 변경 방법](../단계별-학습/1단계-Unity-기초.md#scene-rename-guide) |

- **ConceptTest_Variable, ConceptTest_Array, …** (영문 씬 이름): 각 개념 **실습만** 단계적으로 테스트. **왔다 갔다 없이** 한 문서에서 환경 설정 + 실습 완료.  
- **Tetris** 씬: 1~8단계 **게임 만들기** 진행. **매 단계마다** 작업 씬이 **Tetris**인지 **재차 확인**하세요. **Tetris가 아니면** (SimpleScene, SampleScene 등) **반드시 Tetris로 이름을 변경**한 뒤 진행. → [1단계 - 씬 이름 변경 방법](../단계별-학습/1단계-Unity-기초.md#scene-rename-guide) 참고.

---

## ✅ 정리

1. **Scenes** 폴더를 한 번 준비해 두기 (1단계).
2. **개념 실습**할 때는 **해당 개념 문서**만 연다. 문서 맨 앞의 **"이 실습 전용 씬 만들기"**를 따라 **그 실습 전용 씬**을 만들거나 연다.
3. **같은 문서**에서 **수행 과정**을 따라 **그 씬에서만** 스크립트 작성·실행·확인. 다른 문서로 이동하지 않는다.
4. **테트리스 게임**을 할 때는 **게임용 씬**을 열어서 1~8단계 진행.  
   - **작업 씬 제차 확인**: **매 단계마다** 열린 씬 이름이 **Tetris** (또는 TetrisGame)인지 **다시 한 번** 확인하세요. **Tetris가 아니면** (SimpleScene, SampleScene, Untitled 등) **반드시 Tetris로 이름을 변경**한 뒤 작업하세요. → [1단계 - 씬 이름 변경 방법](../단계별-학습/1단계-Unity-기초.md#scene-rename-guide) 참고.

이렇게 **개념별 전용 씬**으로 나누고, **각 실습 문서에 환경 설정이 포함**되어 있으므로, 왔다 갔다 없이 단계적으로 진행할 수 있습니다.
